# Spendly Profile Page Specification

## 1. Overview

Implement a complete **Profile Page** for the Spendly application.

The Profile Page serves as a personal dashboard where users can view their account information, spending statistics, and recent transaction history.

The design should follow a modern finance dashboard layout with clean cards, clear typography, and responsive behavior across all devices.

---

## 2. Goal

The Profile Page should allow authenticated users to:

- View their profile information
- See total spending statistics
- View total transaction count
- Identify their top spending category
- Review recent transactions

This milestone is **read-only**.

Profile editing functionality is **not required**.

---

## 3. Route

### URL

```text
/profile
```

### Access Control

- Authenticated users can access the page.
- Unauthenticated users must be redirected to:

```text
/login
```

---

## 4. Page Structure

The page consists of three major sections:

### A. Profile Header Card

### B. Statistics Cards

### C. Recent Transactions Table

# 6. Statistics Section

Display three summary cards below the profile header.

Cards should appear in a single row on desktop.

---

## Card 1: Total Spent

### Label

```text
TOTAL SPENT
```

### Value

Example:

```text
₹3,800.00
```

### Calculation

```python
total_spent = sum(expense.amount for expense in user_expenses)
```

---

## Card 2: Transactions

### Label

```text
TRANSACTIONS
```

### Value

Example:

```text
8
```

### Calculation

```python
transaction_count = len(user_expenses)
```

---

## Card 3: Top Category

### Label

```text
TOP CATEGORY
```

### Value

Example:

```text
Bills
```

### Calculation

Group expenses by category and return the category with the highest total spending.

Example:

```python
{
    "Food": 1200,
    "Bills": 2000,
    "Shopping": 600
}
```

Result:

```text
Bills
```

# 7. Recent Transactions Section

Display a card below the statistics section.

### Title

```text
Recent Transactions
```

### Table Columns

| Column | Description |
|----------|----------|
| Date | Expense Date |
| Description | Expense Description |
| Category | Expense Category |
| Amount | Expense Amount |

### Example

| Date | Description | Category | Amount |
|--------|--------|--------|--------|
| Apr 08, 2026 | Lunch with colleagues | Food | ₹180 |
| Apr 08, 2026 | Miscellaneous | Other | ₹200 |
| Apr 07, 2026 | New earphones | Shopping | ₹800 |
| Apr 06, 2026 | Movie tickets | Entertainment | ₹500 |
| Apr 05, 2026 | Pharmacy – vitamins | Health | ₹350 |

### Sorting

Transactions must be ordered by:

```python
expense_date DESC
```

Newest transaction first.
